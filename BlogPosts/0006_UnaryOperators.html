Arithmetic Operators

Introduction

A simple arithemtic operation would look like 3 + 4 or 7 * 8 or 1 / 4 - 6 . Here, + - * / are the arithmetic operators.
+ is used for addition, - for substraction, * for multiplication and / for division.

Special Arithmetic Operator

There is also a special arithmetic operator %. This operator is often termed as 'modulo'.
The operation X % Y calculates the remainder when X is divided by Y. 8 % 5 will result in 3. Also, % operator has the same precedence as * and /.

Precedence and Order of calculation

+ operator has same precedence as - operator and * operator has same precedence as / operator.
* and / have higher precedence than + and - . Which means in an operation like 5 + 3 * 7, 3 * 7 will be calculated first and the result will then be added to 5. Also any operation enclosed between () like (3 + 4) * 7 is given a higher precedence. That is 3 + 4 will be calculated first and the result will then be multiplied with 7. In an operation with operators of same precedence, the calculation is done from left to right.

Let's code!

Let's code our simplest javascript calculation program. Open the Chrome browser console window and type the following:

let x = 15;
let y = 10;
console.log("Result of x + y is : "+ x + y);
console.log("Result of x - y is : "+ x - y);
console.log("Result of x * y is : " + x * y);
console.log("Result of x / y is : "+ x / y);
console.log("Result of x % y is : " + x % y);

Output

Hit enter key and see what happens. Wait something is wrong. The results for addition and substraction don't seem right. Right? 15+10 should 25 and not 1510. Also, 15-10 should be 5 and not NaN. The reason is we have a string within dounle quotes and whenever we have a string, the whole operation is converted to string so basically what we did was 'concatenating' "15" and "10" and not adding those. To fix it, we could have written

console.log("Result of x + y is : "+ (x + y));

and output would be fine. Reason is () has higher precedence, so operation between parenthesis was performed first and later converted to string. Similarly, the output of x - y operation is NaN(Not a Number) because + and - operators have same precedence so basically what we're trying to do is subtracting a number from a string which doesn't make much sense. Hence NaN. Again, we could fix it by writing

console.log("Result of x - y is : "+ (x - y));

and the result would be 5, which is correct. Rest operations produce correct output as operators * / % have higher precedence than + operator.
