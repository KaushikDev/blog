Comments

Introduction

Comments are used to add hints, notes, suggestions, or warnings to JavaScript code. To make other programmers(or even future self) aware of what a particular bit of code does, comment can be used in a variety of ways. Like adding a //ToDo for future feature request and etc. They can also be used to disable code to prevent it from being executed. They can prove a to be a valuable debugging tool. There are two ways of adding comments to your code. These are :

Single Line Comment

Single line comment can be made using two forward slashes or //. Any bit of code written after the // will be non-executable. E.g.

// This is a singe line comment. // Following program returns the sum of two numbers. function(a, b){ return a + b; }//sum of a and b will be returned.

Multi line comment

A multi-line comment can be made using the starting forward slash and asterisk i.e. /* and ending asterisk and forward slash i.e. */ . Any piece of information between /* and */ is considered non-executable. E.g.

/* This is a multi-line comment that goes on for four lines of code.*/

Multi-line comment can also be used as a single line comment. E.g.

/* This is a single line comment too */

VS Code(IDE): Comment Trick(windows)

To comment, use ctrl+k+c . To uncomment, use ctrl+k+u .

Conclusion

It is nice to know about these little tricks of the trade and comments are certainly one of these tricks. Try commenting your own piece of code and see whether it gets included or excluded in executionable code. Until next time. Happy coding.
